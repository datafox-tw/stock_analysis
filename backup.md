1. 2026年版本我的核心 Bug 修復 (針對 2023 年 Flask 版本)

- 修復資料解析錯誤：原先 `app.py` 在不同路由（Route）之間傳遞資料時，會將 Python 的 List 轉成字串。這導致在後端處理時，迴圈會抓到單個字元而不是數值（例如 ma[i] 變成字串中的一個括號或逗點），所以我重寫parser維持原本的手動切割風格但確保邏輯正確。(也就是每次問第二個問題時就會爆炸的問題獲得解決)
- 防止索引越界 (IndexError)：修復了range(len(data)) 的範圍問題。原本的代碼會嘗試讀取 i+1 的資料，我將範圍調整為 len(data)-1。
- 優化股票代碼讀取：原本的   `stocklist.csv`， 解析器會忽略編號小於 1000 的股票（如 0050），我放寬了限制並增加了更強健的錯誤排除邏輯。
- 跨平台路徑處理：將 Windows 特有的路徑分隔符 \\ 改為通用的處理方式，確保在 Mac 或 Linux 上也能順利運行。(不然我本來用mac跑這個python程式都會崩潰)
- 前端修正：修復了 index.html 中靜態檔案（CSS）的連結方式，以及一些隱藏輸入欄位（Hidden Input）的名稱對應錯誤。
